# 3. ES6+ (Sangat Wajib)

ES6 (ECMAScript 2015) membawa revolusi besar dalam penulisan sintaks JavaScript. Ini adalah "bahasa gaul" yang wajib dikuasai untuk memahami framework modern.

## 1. Arrow Function
Bentuk fungsi yang lebih ringkas dan memiliki sifat `this` yang berbeda dari fungsi biasa.

```js
// Fungsi Klasik
function sapa(nama) {
  return "Halo " + nama;
}

// Arrow Function
const sapaArrow = (nama) => {
  return "Halo " + nama;
};

// Arrow Function (Shorthand / 1 Baris)
const sapaCepat = (nama) => "Halo " + nama;
```

## 2. Destructuring
Teknik "membongkar" isi Array atau Object ke dalam variabel terpisah.

### Object Destructuring
```js
const user = { nama: "Dio", umur: 21, asal: "Inggris" };

// Kuno: const nama = user.nama;
// Modern:
const { nama, umur } = user;
console.log(nama); // "Dio"
```

### Array Destructuring
```js
const rgb = [255, 0, 128];
const [red, green, blue] = rgb;

console.log(red); // 255
```

## 3. Spread Operator (`...`)
Membongkar isi array/object untuk disebarkan ke array/object lain. Sering dipakai untuk *copy* atau *merge*.

```js
// Array Spread
const buah = ["Apel", "Jeruk"];
const keranjang = [...buah, "Mangga"]; // ["Apel", "Jeruk", "Mangga"]

// Object Spread
const profil = { nama: "Jojo" };
const profilLengkap = { ...profil, level: 99 }; // { nama: "Jojo", level: 99 }
```

## 4. Rest Parameter (`...`)
Sintaksnya sama seperti Spread, tapi fungsinya kebalikan: **Membungkus** banyak sisa argumen menjadi satu Array.

```js
function jumlahkanSemua(...angka) {
  // 'angka' otomatis menjadi array [1, 2, 3, 4]
  return angka.reduce((total, n) => total + n, 0);
}

console.log(jumlahkanSemua(1, 2, 3, 4)); // 10
```

## 5. Default Parameter
Memberi nilai bawaan pada parameter fungsi jika tidak ada argumen yang dikirim.

```js
function sapa(nama = "Anonim") {
  console.log("Halo " + nama);
}

sapa("Agung"); // "Halo Agung"
sapa();        // "Halo Anonim"
```

## 6. Object Shorthand & Enhanced Literal
Jika nama *key* dan variabel sama, kita bisa menulisnya sekali saja.

```js
const merk = "Toyota";
const tipe = "Supra";

// Kuno: { merk: merk, tipe: tipe }
// Modern (Shorthand):
const mobil = { merk, tipe }; 

console.log(mobil); // { merk: "Toyota", tipe: "Supra" }
```

## 7. Optional Chaining (`?.`)
Sangat krusial saat membaca data API! Digunakan untuk mencegah *Error* jika properti bersarang (nested) ternyata tidak ada (`undefined`).

```js
const user = {
  id: 1,
  // profil tidak ada
};

// Kuno: Rawan Error "Cannot read properties of undefined"
// console.log(user.profil.alamat); // ❌ ERROR APP CRASH

// Modern (Optional Chaining)
console.log(user.profil?.alamat); // ✅ Aman, hanya output: undefined
```

## 8. Nullish Coalescing (`??`)
Menetapkan nilai *fallback/default* **HANYA JIKA** variabel bernilai `null` atau `undefined` (Bukan `0` atau string kosong `""`).

```js
let stokKosong = 0;

// Pakai OR (||) -> Menganggap 0 itu false
let label1 = stokKosong || "Habis"; // Output: "Habis" (SALAH!)

// Pakai Nullish (??) -> 0 dianggap nilai valid
let label2 = stokKosong ?? "Habis"; // Output: 0 (BENAR!)
```

## Arrow Function `this` Binding
Arrow function **tidak memiliki `this` sendiri** — ia mengambil `this` dari scope luar (lexical `this`). Function biasa memiliki `this` yang dinamis (tergantung cara pemanggilan).

```js
const obj = {
  nama: "Budi",
  // Function biasa → this = obj (pemanggil)
  sapaBiasa: function () {
    console.log("Halo " + this.nama);
  },
  // Arrow → this = scope luar (window/global)
  sapaArrow: () => {
    console.log("Halo " + this.nama);
  }
};

obj.sapaBiasa(); // "Halo Budi" ✅
obj.sapaArrow(); // "Halo undefined" ❌ (this bukan obj)
```

## Kapan TIDAK Pakai Arrow Function
1. **Method object** — butuh `this` yang merujuk ke object itu sendiri
2. **Event handler DOM** — `this` di function biasa = element yang diklik, di arrow = `window`
3. **Constructor** — arrow tidak bisa dipakai dengan `new`

```js
// ❌ Arrow sebagai constructor — ERROR
// const Person = (nama) => { this.nama = nama; };
// new Person("Budi"); // TypeError: Person is not a constructor

// ✅ Gunakan function biasa
function Person(nama) {
  this.nama = nama;
}
```

## Destructuring Lanjutan
```js
const user = {
  nama: "Budi",
  alamat: {
    kota: "Jakarta",
    kodepos: "10110"
  }
};

// Nested destructuring
const { alamat: { kota } } = user;
console.log(kota); // "Jakarta"

// Default value
const { umur = 25 } = user;
console.log(umur); // 25 (karena user.umur tidak ada)

// Aliasing (ganti nama variabel)
const { nama: userName } = user;
console.log(userName); // "Budi"
// console.log(nama);   // ❌ Error — nama tidak terdefinisi
```

## `Map` dan `Set`
### Map — key-value dengan key tipe apa saja
```js
const map = new Map();
map.set("nama", "Budi");
map.set(1, "satu");
map.set({ id: 1 }, "data object");

console.log(map.get("nama")); // "Budi"
console.log(map.has(1));      // true
console.log(map.size);        // 3
map.delete(1);

// Iterasi Map
for (const [key, value] of map) {
  console.log(`${key}: ${value}`);
}
```

### Set — koleksi nilai unik
```js
const set = new Set([1, 2, 2, 3, 3, 3]);
console.log(set);      // Set { 1, 2, 3 } (duplikat otomatis dibuang!)
console.log(set.has(2)); // true
console.log(set.size); // 3
set.add(4);
set.delete(1);

// Menghilangkan duplikat dari array
const unik = [...new Set([1, 1, 2, 2, 3])]; // [1, 2, 3]
```

## `WeakMap` & `WeakSet`
Mirip Map/Set, tetapi **key harus object** dan tidak mencegah garbage collection. Cocok untuk data tambahan yang tidak boleh menghalangi object utama dihapus dari memori.

```js
const wm = new WeakMap();
let obj = { id: 1 };
wm.set(obj, "metadata");

obj = null; // obj siap di-garbage-collect, entry di WeakMap juga hilang otomatis
```

## Tagged Template Literal
Fungsi yang dipanggil dengan template literal untuk memproses string dan interpolasi.

```js
function highlight(strings, ...values) {
  return strings.reduce((result, str, i) =>
    `${result}${str}<strong>${values[i] || ""}</strong>`, ""
  );
}

const nama = "Budi";
const umur = 25;
console.log(highlight`Nama: ${nama}, Umur: ${umur}`);
// Output: "Nama: <strong>Budi</strong>, Umur: <strong>25</strong>"
```

## Computed Property Name
```js
const key = "nama";
const obj = { [key]: "Budi", umur: 20 };

console.log(obj.nama); // "Budi"
console.log(obj);      // { nama: "Budi", umur: 20 }

// Dinamis dengan ekspresi:
const idx = 1;
const data = { ["item" + idx]: "Data ke-1" };
console.log(data.item1); // "Data ke-1"
```

## Optional Chaining Variations
```js
const user = { nama: "Budi", hobi: ["Baca", "Lari"], sapa: () => "Halo!" };

// Object property access (sudah dibahas)
console.log(user?.profil?.alamat); // undefined

// Array access
const arr = null;
console.log(arr?.[0]); // undefined (tidak error!)

// Function call
const fn = null;
// fn();              // ❌ TypeError
console.log(fn?.());  // ✅ undefined

// Bracket notation
console.log(user?.["hobi"]?.[1]); // "Lari"
```
